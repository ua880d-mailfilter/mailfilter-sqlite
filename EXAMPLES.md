# EXAMPLES.md

## Overview

This document provides real-world examples generated by the mailfilter rulegen system.

It demonstrates how raw header data is transformed into actionable rules.

---

## 1. Fake Brand Detection (Phishing)

Typosquatting and brand impersonation are detected automatically.

### Example

    from_domain: arnazon-billing.example
    brand_match: amazon

### Generated Rule

    # Fake brand (Amazon impersonation)
    SCORE +20="^(From|Reply-To|Return-path):.*arnazon-billing\.example"

---

## 2. Campaign Detection

Multiple emails are grouped into campaigns based on shared characteristics.

### Example

    from_domain: amazon-login-check.example
    received_host: mx-relay167-hz1.antispameurope.com

### Generated Rules

    # Campaign correlation (domain + infrastructure)
    SCORE +20="^(From|Reply-To):.*amazon-login-check\.example"
    SCORE +20="^(From|Received):.*mx-relay167-hz1\.antispameurope\.com"

---

## 3. Promotional Spam (Price + Urgency)

Typical spam patterns combining urgency and pricing.

### Example Subjects

    Final Countdown! 72Hrs Only - Smart Watch $19.99
    Last 48Hrs Sale - Ultrasonic Machine Only $39.99

### Generated Rules

    # Hard spam (promo campaign)
    DENY="^From:.*cheap-clearance\.example"

    # Subject pattern scoring
    SCORE +15="^Subject:.*(48Hrs|72Hrs|Countdown|Today Only).*"

---

## 4. Infrastructure-Based Detection

Repeated use of known mail relays or spam infrastructure.

### Example

    received_host: relay.badmail.example

### Generated Rule

    SCORE +20="^(From|Received):.*relay\.badmail\.example"

---

## 5. False Positive Protection (ALLOW Proximity)

Legitimate-looking content prevents aggressive blocking.

### Example

    from_domain: dhl-delivery-center.example
    reason: ALLOW proximity

### Generated Rule

    # No DENY due to potential false positives
    SCORE +20="^(From|Reply-To):.*dhl-delivery-center\.example"

---

## Summary

These examples illustrate:

- Detection of phishing and brand impersonation
- Campaign clustering across multiple messages
- Recognition of spam patterns
- Infrastructure-based correlation
- Built-in false positive protection

The system combines all of these aspects to generate safe and effective rules.
